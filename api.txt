Rating Integration Guide (Public & Admin Routes)
These instructions cover the newly added public and admin routes for the rating module.
1. Public: Get Class Ratings
Endpoint: GET /api/v1/ratings/class/:classId
Auth: None (Public)
Description: Use this to show a list of reviews on a specific Class details page.
Example: /api/v1/ratings/class/69e41bfeaaf28b4527e8329f?page=1&limit=5
2. Public: Get Teacher Ratings
Endpoint: GET /api/v1/ratings/teacher/:tutorId
Auth: None (Public)
Description: Use this to show all reviews for a specific teacher on their profile page.
Example: /api/v1/ratings/teacher/69e41bb9e030b114c94bb70a?page=1&limit=10&sort=-rating
3. Admin: Manage All Ratings
Endpoint: GET /api/v1/ratings/admin
Auth: Required (ADMIN or SUPER_ADMIN)
Description: An admin-only view to see every rating in the system. Supports all filters (tutor, student, class) and full pagination.
Example: /api/v1/ratings/admin?limit=20&page=1
Pagination & Sorting (Common Params)
All the above routes support these query parameters:
Parameter	Type	Default	Description

page
	Number	
1
	The current page number.

limit
	Number	
10
	Number of items per page.

sort
	String	
-createdAt
	Sort field (e.g., rating for lowest first,
-rating for highest first).
